用法

最简单的用法：首先设置配置(例如autoescaping)，然后渲染一个字符串

一般并不会用renderString，而是使用render来直接渲染一个独立的模板文件。使用这种方式你还可以继承或引入模板。使用这种方式首先需要在nunjucks的第一个参数中配置文件的路径

在node中，’views’是相对当前工作路径的相对路劲。在浏览器中，它是一个相对的URL。如果要指定绝对路径，使用’/views’

如果使用的是express,可以将express应用程序直接传入nunjucks

上面的 API 适用于 node 端和浏览器端 (express 只适用于 node 端)，在 node 端 nunjucks 从文件系统加载模板，在浏览器端通过 http 加载模板。

在循环中，可以获取一些特殊的变量

  • loop.index:循环的当前索引(从1开始)
  • loop.index0:循环的当前索引(从0开始)
  • loop.revindex: 索引数(从后往前) (1 indexed)
  • loop.revindex0: 索引数(从后往前) (0 based)
  • loop.first: 是否是第一个
  • loop.last: 是否是最后一个
  • loop.length: 总数

extends

extends用来制定模板继承，被指定的模板作为基本的父模板。

你可以将继承的模板设为一个变量，这样就可以动态指定继承的模板。这个变量既可以是个指向模板文件的字符串，也可以是个模板编译后所生成的对象(需要添加上下文环境)。因此你可以通过设置上下文变量，从而在渲染时动态地改变所要继承的模板。

事实上，extends接受任意表达式，只要它最终返回一个字符串或一个编译过的模板对象

blocks

block定义了一个模板片段，通过名字来进行标识，用来实现模板继承。基本的父模板会指定一个区块，子模板会用新内容覆盖它。

你也可以通过循环来定义一个模板片段

子模板会覆盖item片段并改变它的内容

super可以将父模板的内容保留并渲染出来

include

include引入其他的模板，如果某些模板已经继承了其他的模板，而又需要在这些模板中共享一些小模板，include将会非常有用

你还可以在循环的中间引入模板

这一点可以帮助我们把模板切分成更小的部分，从而使得在浏览器上，当我们需要改变页面时，我们可以渲染这些小部分的模板，而非一整个的大的模板。

include 可以接受任意表达式，只要它最终返回一个字符串或是模板所编译成的对象

如果我们不希望在模板不存在时抛出异常，使用ignore missing选项来阻止抛出异常

被包含的模版自身可以扩展(extends)另一个模版（因此你可以让一系列相关联的模版继承同一种结构）。 一个被包含的模版并不会改变包含它的模版的区块结构，它有一个分离的继承树和块级命名空间。换言之， 在渲染时，include并不 不是 将被包含模版代码拉取到包含它的模版中的预处理器。相对的，它对被 包含的模版触发了一次的分离渲染，然后再将渲染的结果引入
